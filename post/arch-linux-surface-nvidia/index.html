在 ArchLinux/SurfaceBook2 上安装配置 Nvidia 显卡驱动

2020-06-20
linux

文章目录

  • 魔改内核
  • 显卡设置

魔改内核

为了启用 SB2 的一些独有的特性如触摸屏和手写笔等，首先要把默认的内核换成魔改过后的内核，这里推荐 linux-surface/linux-surface。

将步骤简单归纳如下：

# DKMS 模块
sudo pacman -S dkms acpi acpi_call-dkms
# 添加 linux-surface 仓库
wget -qO - https://raw.githubusercontent.com/linux-surface/linux-surface/master/pkg/keys/surface.asc \
    | sudo pacman-key --add -
sudo pacman-key --finger 56C464BAAC421453
sudo pacman-key --lsign-key 56C464BAAC421453
sudo sed -i '$a\[linux-surface\]\nServer = https:\/\/pkg\.surfacelinux\.com\/arch' /etc/pacman.conf
sudo pacman -Sy
# 安装内核和固件
sudo pacman -S linux-surface-headers linux-surface surface-ipts-firmware
# 从 AUR 里安装一些其它固件和工具
pikaur -S update-grub aic94xx-firmware wd719x-firmware libwacom-surface surface-control surface-dtx-daemon
sudo update-grub # 更新 grub，下次开机的时候选 linux-surface 这个内核
sudo systemctl enable surface-dtx-daemon.service

显卡设置

  • 先装一些包
1

sudo pacman -S nvidia-dkms nvidia-settings nvidia-utils opencl-nvidia lib32-nvidia-utils lib32-opencl-nvidia bumblebee
  • 把 nouveau 加入黑名单

在 /etc/modprobe.d/blacklist-nouveau.conf

1
2

blacklist nouveau
options nouveau modeset=0
  • 这个魔改内核默认会禁用 nvidia 显卡，这里把它设置成默认开启

在 /etc/modprobe.d/dgpu.conf

1

options surface_sam_hps dgpu_power_init=1
  • 集成显卡用英特尔驱动

在 /etc/X11/xorg.conf.d/20-intel.conf

1
2
3
4
5
6
7

Section "Device"
   Identifier  "Intel Graphics"
   Driver      "intel"
   Option      "TripleBuffer" "true"
   Option      "TearFree"     "true"
   Option      "DRI"          "true"
EndSection
  • 独显用刚刚安装的闭源驱动

在 /etc/X11/xorg.conf.d/20-nvidia.conf

1
2
3
4
5

Section "Device"
    Identifier     "Nvidia Card"
    Driver         "nvidia"
    VendorName     "NVIDIA Corporation"
EndSection
  • 自动加载英伟达模块

在 /etc/modules-load.d/nvidia.conf

1

nvidia
  • 设置 bumblebee 默认使用的驱动

在 /etc/bumblebee/bumblebee.conf 中，把 Driver= 改成 Driver=nvidia

  • 启动系统服务
1
2

sudo usermod -a -G bumblebee your_user_name
sudo systemctl enable bumblebeed.service

重启计算机（注意在开机的时候，进入 advanced options for arch linux，然后选 linux-surface 这个内核），执行下面这条命令，看看是否配置成功
